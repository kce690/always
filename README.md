# 恒聊助手

面向中文用户与 QQ 场景的个人智能体项目，目标是把“能回复”推进到“有长期记忆、能主动互动、状态连续可追踪”。

## 项目简介

这个项目要解决的问题，不是单轮问答，而是长期对话中的“关系断档”和“状态断档”：

- 聊天记录很多，但下一次对话常常忘记上下文。
- 机器人能被动回复，但缺少主动提醒和任务推进能力。
- 多渠道沟通后，身份、状态、任务进度难以统一。

与常见聊天机器人相比，当前仓库更强调以下方向：

- 面向中文真实聊天场景：回复策略、输入处理、弱输入识别更贴近日常聊天。
- 接入本地化沟通渠道：包含 QQ 在内的多渠道消息接入与路由。
- 长期陪伴导向：通过记忆归档、状态文件、定时与心跳机制，保持持续性。

## 核心能力

### 1) 本地可运行的 Agent CLI
提供初始化、单轮对话、交互会话、状态查看等命令入口，可用于日常调试和本地使用。

### 2) QQ 与多渠道接入
已具备 QQ、Telegram、Discord、飞书、钉钉、Slack、企业微信、Matrix、Email、WhatsApp、Mochat 等渠道模块，并有对应测试覆盖。

### 3) 长期记忆归档
会话内容可归档到长期记忆与历史日志，支持上下文压缩与失败兜底归档，减少长期对话中的信息丢失。

### 4) 状态连续系统
内置状态文件与生活日志机制，支持状态推进、离线补偿和状态字段校正，用于维持连续的人设与互动节奏。

### 4.1) 预历史（Prehistory）事件引导
首次实例启动时，系统会进行一次“可重建的预历史引导”：

- 不是写一段静态人设故事，而是生成结构化历史事件流。
- 事件先写入不可变原始日志，再进入记忆索引与遗忘模型。
- 旧事件会按时间戳自然衰减，形成“细节淡化、gist 保留、部分遗忘”的真实效果。
- 当前 `LIFESTATE` 快照由历史事件回放推导，不是独立硬编码。
- 生成过程带种子与版本元数据，支持审计、重建与显式再引导（默认不破坏已有历史）。

### 5) 主动交互能力
支持定时任务与心跳检查：可周期读取任务文件、触发执行并向可达渠道回传结果。

### 6) 工具与扩展能力
支持 Web 搜索、文件系统操作、命令执行限制、MCP 外部工具接入，以及可打包的技能目录结构。

### 7) 多实例与隔离运行
支持按配置文件启动多个实例，运行目录、媒体目录、定时任务目录可按实例隔离。

### 8) 工程化基础
包含 Docker 运行配置与较完整测试集，覆盖渠道、配置迁移、记忆、任务调度、工具校验等关键模块。

- 从 0 到 1 的个人智能体实践价值：覆盖消息接入、记忆、状态、主动任务、扩展工具的完整闭环。

## 快速开始

说明：仓库中存在可执行入口；为了避免环境差异，这里先自动发现本机安装后的 CLI 名称，再执行。

### 1) 安装（开发模式）

```powershell
python -m pip install -e .
```

### 2) 自动发现 CLI 入口名称（PowerShell）

```powershell
$env:BOT_CMD = python -c "import importlib.metadata as m; print(next(ep.name for ep in m.entry_points(group='console_scripts') if ep.value.endswith('cli.commands:app')))"
Write-Host "CLI = $env:BOT_CMD"
```

### 3) 初始化配置与工作区

```powershell
& $env:BOT_CMD onboard
```

初始化后，终端会打印配置文件与工作区路径。按提示填写至少一个可用模型提供方的 `apiKey`、`model`、`provider`。

### 4) 本地交互

```powershell
& $env:BOT_CMD agent
```

### 5) 启动网关（用于消息渠道）

```powershell
& $env:BOT_CMD gateway
```

### 6) 查看渠道状态

```powershell
& $env:BOT_CMD channels status
```

如果你使用 Linux/macOS，可把上面的 CLI 名称发现步骤改写为等价 shell 写法；其余流程一致。
